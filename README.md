# CUBOT: 自动魔方复原机器人 🤖

本项目是一个低成本、高精度的自动化魔方复原系统。系统摒弃了昂贵的工业相机方案，采用普通安卓手机作为视觉传感器，通过 PC 端进行算法推导，并由 Arduino 控制机械结构完成物理复原。

## ⚙️ 系统工作流程
系统整体分为感知、决策和执行三个核心环节，通过 Python 主程序进行统一调度：
1. **指令触发**：Arduino 初始化完成后，通过串口向 PC 发送就绪信号。
2. **视觉采集**：PC 端通过 ADB 协议，自动控制连接的安卓手机拍摄魔方 6 个面的图像。
3. **算法求解**：PC 端利用 OpenCV 提取图像中的 54 个色块信息，并输入 Kociemba 算法，在 5 秒内计算出最少步数的复原序列。
4. **物理执行**：PC 将计算好的动作指令（如 R, U', F2）通过串口下发给 Arduino，Arduino 驱动底部的 8 个舵机完成同步复原。

## 🛠️ 核心贡献
我在本项目中独立负责**软件控制中枢与跨端通信**的开发，主要完成了以下工作：
* **系统通信链路搭建**：编写 Python 脚本封装 ADB 指令，实现了 PC 对手机摄像头的自动化调度；同时设计了 PC 与单片机之间的串口握手协议，打通了全链路的数据传输。
* **异步控制优化**：针对硬件通信容易导致主程序阻塞的问题，使用多线程（Threading）分离了串口读取与视觉计算逻辑，将指令响应延迟控制在 1.0s 以内，保障了动作的连贯性。
* **视觉识别调优**：使用 OpenCV 实现了基于 HSV 色彩空间的颜色提取，并通过调整阈值过滤环境光干扰，使系统在常规光照下的颜色识别准确率达到 90% 以上。

## 📂 仓库结构
* **`Arduino/`**: 包含底层 8 舵机的 C++ 控制代码（负责解析串口指令并输出 PWM 脉冲）。
* **`Python/`**: 项目的主控端代码，包含 OpenCV 视觉识别、Kociemba 算法逻辑以及多线程调度脚本。
* **`Tools/`**: 预置的 Android SDK 平台工具包（ADB），用于实现免配置的手机连接。
